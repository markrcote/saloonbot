## Directory Purposes

**`cardgames/`:**
- Purpose: All server-side game logic and infrastructure; imported only by `server.py` and `cli.py`
- Contains: Game state machine, deck management, player model, casino orchestrator, MySQL data layer, NPC strategies
- Key files: `blackjack.py` (core game), `casino.py` (Redis event loop), `database.py` (persistence)

**`wwnames/`:**
- Purpose: Standalone name generator; imported only by `bot.py` for the `/wwname` slash command
- Contains: `WildWestNames` class and three flat text files with name lists
- Key files: `wwnames.py`, `names/M.txt`, `names/F.txt`, `names/S.txt`

**`.planning/codebase/`:**
- Purpose: Architecture maps generated by GSD tooling; consumed by planning and execution commands
- Contains: ARCHITECTURE.md, STRUCTURE.md, and other analysis docs
- Generated: Yes (by `/gsd-map-codebase`)
- Committed: Yes

**`.github/workflows/`:**
- Purpose: CI/CD automation (linting, tests, image builds)
- Generated: No
- Committed: Yes

## Key File Locations

**Entry Points:**
- `bot.py`: Discord bot process ‚Äî slash commands, Redis subscription, game rendering
- `server.py`: Game server process ‚Äî instantiates `Database` and `Casino`, starts blocking loop
- `cli.py`: Standalone async REPL for testing game logic against a live server

**Configuration:**
- `.flake8`: Linting rules (max-line-length = 120)
- `.python-version`: Python version for pyenv
- `requirements.txt`: All Python dependencies
- `compose.yml`: Production service definitions

**Core Game Logic:**
- `cardgames/blackjack.py`: `Blackjack` class, `HandState` enum, `Action` constants, serialization helpers
- `cardgames/casino.py`: `Casino` class ‚Äî Redis listener, game registry, NPC management
- `cardgames/card_game.py`: `CardGame` base, `Card` class, `CardGameError`
- `cardgames/database.py`: `Database` class ‚Äî all MySQL operations

**Player Abstractions:**
- `cardgames/player.py`: `Player`, `PlayerRegistry`, module-level `registry` singleton
- `cardgames/npc_player.py`: `NPCPlayer` abstract base class
- `cardgames/simple_npc.py`: `SimpleBlackjackNPC` ‚Äî basic strategy implementation

**Testing:**
- `test.py`: Unit tests (run with `python test.py`)
- `test_e2e.py`: End-to-end tests using real Redis + MySQL (run with `./run-e2e-tests.sh`)

## Naming Conventions

**Files:**
- Snake case for all Python modules: `blackjack.py`, `card_game.py`, `npc_player.py`
- Test files at root level: `test.py` (unit), `test_e2e.py` (e2e)
- Shell scripts use hyphens: `run-e2e-tests.sh`, `dev-bot.sh`
- Docker Compose files use `compose.{variant}.yml` naming

**Classes:**
- PascalCase throughout: `Blackjack`, `Casino`, `CardGame`, `NPCPlayer`, `SimpleBlackjackNPC`, `WildWestNames`
- Exceptions suffixed with `Error`: `CardGameError`, `NotPlayerTurnError`, `InsufficientFundsError`
- Enums PascalCase with UPPER values: `HandState.WAITING`, `GameState.ACTIVE`

**Functions and Methods:**
- Snake case: `new_game()`, `start_betting()`, `dealer_turn()`, `to_dict()`, `from_dict()`
- Private methods prefixed with `_`: `_connect()`, `_save_game()`, `_tick_waiting()`, `_validate_action()`
- Tick sub-methods follow pattern `_tick_{state_name}()`: `_tick_betting()`, `_tick_playing()`, etc.

**Variables:**
- Snake case: `game_id`, `player_name`, `request_id`, `time_last_event`
- Environment variables: UPPER_SNAKE in both env and Python constants

**Redis Topics:**
- Lowercase with underscores: `"casino"`, `"casino_update"`, `"game_updates_{game_id}"`

## Where to Add New Code

**New Game Type (e.g., poker):**
- Game logic: `cardgames/poker.py` ‚Äî subclass `CardGame` from `card_game.py`
- Register in casino: Add handling in `cardgames/casino.py`
- Bot commands: Add slash commands in `bot.py` or a new cog file

**New NPC Strategy:**
- Implementation: `cardgames/{strategy_name}_npc.py` ‚Äî subclass `NPCPlayer`
- Register: Add entry to `NPC_TYPES` dict in `cardgames/casino.py`

**New Bot Slash Command:**
- Location: `bot.py` in `BlackjackCog` class using `@nextcord.slash_command(guild_ids=GUILD_IDS)`
- Or create a new Cog and add it with `bot.add_cog()`

**New Database Table/Operation:**
- Location: `cardgames/database.py` ‚Äî add method to `Database` class; add `CREATE TABLE IF NOT EXISTS` in `_init_database()`

**New Name Data:**
- Location: `wwnames/names/` ‚Äî add to `M.txt`, `F.txt`, or `S.txt`; or extend `WildWestNames` in `wwnames/wwnames.py`

**Tests:**
- Unit tests: `test.py`
- E2E tests: `test_e2e.py`
